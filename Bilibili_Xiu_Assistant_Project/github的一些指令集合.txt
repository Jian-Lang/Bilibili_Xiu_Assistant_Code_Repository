上传代码到github的方法：
1.首先设置好自己的ssh
2.而后开始上传：
（1）首先登录github，新建一个仓库，复制仓库的链接（建议保存SSH版本的链接）
（2）在电脑中任意位置新建一个文件夹（该文件夹路径不要有中文）
（3）此时打开本地git的Bash，并用cd代码进入该新建文件夹内
（4）运行指令：git clone 仓库链接（第一步的链接复制在这里即可）
（5）此时在第二步的文件夹中，会出现一个新的文件夹，名称就是第一步新建仓库的名称
（6）复制你要上传的代码的文件，并黏贴在在第五步的新文件夹中
（7）于是再通过cd进入第五步的新文件夹，而后运行指令：git status
此时会出现一些报错字段，先忽视掉即可
（8）运行指令：git add ./
（9）运行指令：git commit -m "具体的评论内容，可以是跟你的代码相关的描述信息"
（10）运行指令：git push -u origin master
Finished!
注意第四步clone仓库到本地可能会出现超时错误，此时把第一步的链接切换到SSH/HTTP即可

git的其他操作指令集合
1.查看提交的记录（主要是看自己写的commit）
git log

2.git add之后，如果有更改内容，可以查看具体的更改
git diff

3.撤销本次的修改，即恢复文件到本次修改前的状态
git checkout -- .

4.版本回退（重要）
分两步走：
（1）使用git log，查看历次的版本情况，在黄色的commit行中，commit后面跟着的是版本的序号，取前七位，作为版本号
（2）再键入指令git reset --hard 七位版本号 即可完成任何版本的回退
注意：回退到旧版本的时候，再键入git log时只能看到该旧版本及之前的commit状态和相关的版本和版本号，看不到该版本后面的版本（已被删除）

5.版本恢复（重要）
分两步走：
（1）使用git reflog，查看包括版本回退（删除的版本）的所有版本的commit和版本号的信息，于是找到想要恢复的版本号
（2）再键入指令git reset --hard 七位版本号 即可完成任何版本的恢复（即使已被回退删除）

6.删除本地仓库的文件
这个操作的发生时间是在本地git仓库文件夹中新建了一个文件，但是还未执行add操作的时候（若已执行git add，则本项操作无效）
键入指令 git clean -xf 即可

git多人操作指南
（1）首先fork团队的仓库到自己的账户中（获取团队仓库的镜像）
（2）使用克隆指令，克隆自己的镜像仓库到本地
（3）拉取团队的最新仓库（这一步可选择执行，执行取决于团队仓库是否发生了更改，若无更改则无需本步操作）:
  首先输入指令：git remote -v ，查看自己是否有团队项目的upstream，
  若无，则继续执行操作 git remote add upstream 团队项目地址（这个地址是项目拥有者的ssh地址）
  而后再次输入 git remote -v ，此时应该出现upstream
  而后输入指令 git fetch upstream 获取团队的最新版本
  最后一步执行合并操作：git merge upstream/分支名称（该名称决定想要拉取团队仓库的分支是哪一个，例如拉取master分支，则git merge upstream/master）
（4）拉取后，在本地可以自由开发，开发之后使用push ，push到自己的镜像仓库
（5）想要和团队合并时，在github上进行发起分之合并request即可
